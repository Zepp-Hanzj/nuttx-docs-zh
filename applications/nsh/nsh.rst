========
概述
========

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

**NSH 库**。``apps/nshlib`` 子目录包含 NuttShell (NSH) 库。
该库可以轻松链接以生成 NSH 应用程序（参见 ``apps/examples/nsh`` 示例）。
NSH 库为 NuttX 提供了一个简单的 shell 应用程序。

控制台/NSH 前端
=====================

**NSH 控制台**。使用配置文件中的设置，NSH 可以配置为使用 (1) 串口
stdin/stdout、(2) USB 串口设备（如 CDC/ACM）或 (3) telnet 连接作为
控制台。或者，甚至可以同时使用以上所有方式。支持无限数量的 telnet 会话。

**启动提示符**。当 NSH 启动时，您将在选定的控制台上看到如下欢迎消息：

.. code-block::

  NuttShell (NSH)
  nsh>

如果您使用的是带版本号的 NuttX，欢迎消息还可能包含 NuttX 版本信息。
``nsh>`` 是 NSH 提示符，表示您可以从控制台输入命令。

**USB 控制台启动**。使用 USB 控制台时，启动序列略有不同：
在这种情况下，您需要按 *Enter* 键三次。然后 NSH 提示符将如上所述出现。
这样做的原因如下：

  #. 这确保 USB 连接是稳定的。如果 USB 电缆尚未完全就位，USB 连接可能会
     被建立、断开和重新建立几次。等待 *Enter* 键被按下三次可确保连接稳定。
  #. 连接的建立是一个两步过程：首先，与主机 PC 建立 USB 串口连接。然后在
     主机上启动使用串口接口的应用程序。当主机上建立串口连接时，主机操作
     系统可能会根据主机串口的配置发送多个 *AT* 调制解调器命令。通过等待
     *Enter* 键被连续按下三次，所有这些调制解调器命令将被丢弃，不会被解释
     为 NSH 命令输入。
  #. 类似地，在第二步中当应用程序启动时，可能会有额外的 *AT* 调制解调器
     命令从串口发送出去。大多数串口终端程序都会这样做，除非它们被特别配置
     为抑制调制解调器命令输出。等待 *Enter* 输入消除了来自 (2) 和 (3) 的
     无效命令错误。
  #. 最后，如果 NSH 不等待串口终端程序启动和运行的某种积极指示，那么 NSH
     欢迎消息和初始 NSH 提示符的输出将会丢失。

**扩展命令行编辑**。默认情况下，NuttX 使用简单的命令行编辑器，允许在
``nsh>`` 之后输入命令，仅支持 *退格* 键进行编辑。但是，可以通过在 NuttX
配置文件中设置 ``CONFIG_NSH_CLE=y`` 来选择更完整的命令行编辑器。选择该
选项后，支持以下类似 EMACS 的行编辑命令：

=====================  ================================================
按键绑定               编辑器操作
=====================  ================================================
``^A``                 将光标移动到行首
``^B``                 向左移动一个字符
``^D`` 或 *Del*        删除光标位置的一个字符
``^E``                 将光标移动到当前行末尾
``^F``                 向右移动一个字符
``^H`` 或 *Backspace*  向左删除字符（退格）
``^K``                 删除到行尾
``^U``                 删除整行
=====================  ================================================

命令概述
================

**简单命令、重定向命令和后台命令**。NuttShell (NSH) 是一个简单的 shell
应用程序。NSH 支持以下命令形式：

===============================   ===========================================
简单命令                          ``<cmd>``
带重定向输入的命令                ``<cmd> < <file>``
带重定向输出的命令                ``<cmd> > <file>`` ``<cmd> >> <file>``
带重定向错误的命令                ``<cmd> 2> <file>`` ``<cmd> 2>> <file>``
将 stderr 重定向到 stdout         ``<cmd> > <file> 2>&1``
后台命令                          ``<cmd> &``
带重定向的后台命令                ``<cmd> > <file> &`` ``<cmd> >> <file> &``
===============================   ===========================================

其中：

  * ``<cmd>`` 是后面列出的任何简单命令。
  * ``<file>`` 是文件系统命名空间中任何可读或可写对象的完整或相对路径
    （文件或字符驱动程序）。在本文档中，这些对象将简称为文件。

``nice`` **后台命令**。NSH 以中等优先级 (128) 执行。后台命令可以使用
``nice`` 以更高或更低的优先级执行::

  [nice [-d <niceness>>]] <cmd> [< <file>] [> <file>|>> <file>] [2> <file>|2>> <file>|2>&1] [&]

其中 ``<niceness>`` 是 -20 到 19 之间的任何值，较低（更负）的值对应于
更高的优先级。默认 niceness 为 10。

**每行多个命令**。NSH 将在每个命令行上接受多个命令，每个命令之间用分号
字符 (;) 分隔。

**可选语法扩展** 由于这些功能占用大量资源，它们默认是禁用的。

  -  ``CONFIG_NSH_CMDPARMS``：如果选中，则命令、文件应用程序和 NSH 内置
     命令的输出可以用作其他命令的参数。要执行的实体通过将命令行括在反引号
     中来标识。例如，

     .. code-block:: bash

       set FOO `myprogram $BAR`

     将执行名为 ``myprogram`` 的程序，将环境变量 ``BAR`` 的值传递给它。
     环境变量 ``FOO`` 的值随后被设置为 ``myprogram`` 在 ``stdout`` 上的输出。

  -  ``CONFIG_NSH_ARGCAT``：支持将字符串与环境变量或命令输出连接。例如：

     .. code-block:: bash

       set FOO XYZ
       set BAR 123
       set FOOBAR ABC_${FOO}_${BAR}

     将把环境变量 ``FOO`` 设置为 ``XYZ``，``BAR`` 设置为 ``123``，
     ``FOOBAR`` 设置为 ``ABC_XYZ_123``。如果未选中 ``CONFIG_NSH_ARGCAT``，
     则 FLASH 占用会略微减小，但命令行上只能使用简单的环境变量如 ``$FOO``。

  -  ``CONFIG_NSH_QUOTE``：启用命令中某些字符的反斜杠引用。此选项适用于
     使用 NSH 脚本动态生成新 NSH 脚本的情况。在这种情况下，命令必须被视为
     简单文本字符串，不解释任何特殊字符。``$``、:literal:`\\`、``"`` 等特殊
     字符必须作为测试字符串的一部分保持完整。此选项当前仅在同时选中了
     ``CONFIG_NSH_ARGCAT`` 时可用。

条件命令执行
=============================

还支持 ``if-then[-else]-fi`` 构造，以支持命令的条件执行。这可以从命令行
使用，但主要用于 NSH 脚本中（参见 ```sh`` <#cmdsh>`__ 命令）。语法如下：

.. code-block:: bash

  if [!] <cmd>
  then
    [<cmd> 序列]
  else
    [<cmd> 序列]
  fi

其中 ``<cmd>`` 是 `简单命令 <#command-overview>`__。命令成功值为零被视为真；
非零命令失败值被视为假。```test`` <#cmdtest>`__ 命令通常用于比较。

示例：

.. code-block:: bash

  nsh> if test 1 -eq 1; then echo "1 is equal 1"; else echo "1 is not equal 1"; fi
  1 is equal 1

  nsh> if test 1 -eq 0; then echo "1 is equal 0"; else echo "1 is not equal 0"; fi
  1 is not equal 0

循环
=======

**循环构造**。还支持 ``while-do-done`` 和 ``until-do-done`` 循环构造。
这些可以从命令行使用，但主要用于 NSH 脚本中（参见 ```sh`` <#cmdsh>`__
命令）。

  - ``while-do-done``：只要 ``<cmd>`` 的退出状态为零，就执行 ``[<cmd> 序列]``。
    语法如下：

    .. code-block:: bash

      while <cmd>
      do
        [<cmd> 序列]
      done

  - ``until-do-done``：只要 ``<cmd>`` 的退出状态为非零，就执行 ``[<cmd> 序列]``。
    语法如下：

    .. code-block::

      until <cmd>
      do
        [<cmd> 序列]
      done

其中 ``<cmd>`` 是 `简单命令 <#command-overview>`__。命令成功值为零被视为真；
非零命令失败值被视为假。```test`` <#cmdtest>`__ 命令通常用于比较。

**``break`` 命令**。还支持 ``break`` 命令。``break`` 命令仅在 ``while`` 或
``until`` 循环体中有意义，位于 ``do`` 和 ``done`` 标记之间。如果 ``break``
命令在循环体内执行，循环将立即终止，执行将继续到 ``done`` 标记之后的下一个命令。

当前工作目录
=========================

``cd`` **和** ``pwd``。所有命令的路径参数可以是绝对路径或相对于当前工作目录的
路径。当前工作目录使用 ``cd`` 命令设置，可以使用 ``pwd`` 命令或 ``echo $PWD``
命令查询。

``OLDPWD``  上一个工作目录
