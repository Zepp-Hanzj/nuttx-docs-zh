==============
资源表
==============

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

资源表是定义在共享内存中的关键数据结构，由 RPTUN 驱动提供给 RPTUN 框架。它描述了可用的系统资源。

资源表内容
=======================

VirtIO 设备（vdev 资源）
-----------------------------

定义 VirtIO 设备类型、特性和 Vring 相关参数。

共享内存区域（carveout 资源）
----------------------------------------

定义用于 Virtqueue、数据缓冲区和特殊缓冲区区域的共享内存的物理地址和大小。

共享内存布局
====================

RPTUN 框架中使用的共享内存由各种元素使用，包括资源表、Vrings、Vring 缓冲区、Carveout 等。此共享内存的布局对框架至关重要。

RPTUN 框架依赖驱动在资源表中声明一个连续的共享内存块。RPTUN 在此内存上构建以下结构：

::

  +------------------------------------------+
  |           资源表 (RSC)                    |
  |  +------------------------------------+  |
  |  |  VirtIO 设备 0 描述                |  |
  |  |  (vdev 资源 + carveout)            |  |
  |  +------------------------------------+  |
  |  |  VirtIO 设备 1 描述                |  |
  |  |  (vdev 资源 + carveout)            |  |
  |  +------------------------------------+  |
  |  |  ...                               |  |
  |  +------------------------------------+  |
  +------------------------------------------+
  |              Vrings                       |
  |  +------------------------------------+  |
  |  |  Vring 0 (TX)                      |  |
  |  +------------------------------------+  |
  |  |  Vring 1 (RX)                      |  |
  |  +------------------------------------+  |
  +------------------------------------------+
  |         Carveout（共享堆）               |
  |  （Vring 缓冲区的动态分配）              |
  +------------------------------------------+

资源表 (RSC)
--------------------

描述所有共享资源。其地址直接由驱动提供给 RPTUN 框架。RSC 内部包含一个或多个 VirtIO 设备的描述。每个 VirtIO 设备由一个 vdev 资源和一个 carveout 资源共同表示。

Vrings
------

存储 Virtqueue 的环形缓冲区数据结构。每个 vdev 资源根据标准描述多个 vrings。地址分配规则如下：

- **动态分配**：如果资源表中 vdev 资源的 ``vring.da``（设备地址）字段为 0 或 ANY，RPTUN 框架将自动从 carveout 区域为 Vrings 分配内存块。

- **静态指定**：如果 ``vring.da`` 是有效地址，表示驱动已静态指定 Vrings 的内存位置，框架将直接使用该地址。

Carveout（共享内存堆）
-----------------------------

每个 VirtIO 设备必须对应一个 carveout 资源以提供动态分配的共享内存。RPTUN 框架使用 carveout 区域（减去可能已分配给 Vrings 的部分）初始化独立的共享内存堆。

上层 VirtIO/Vhost 驱动可以通过此堆动态分配和释放在 Vrings 中传输的数据缓冲区（Vring Buffers），实现灵活的内存管理。

地址转换
===================

在 AMP 架构中，不同核可能以不同的物理地址访问同一共享内存块。为确保指针在远程核上有效，当将本地核的指针传递给远端时，需要将地址转换为远端可访问的地址。

RPTUN 框架通过 ``rptun_ops`` 中的 ``get_addrenv()`` 接口获取地址转换表。在适配驱动时，如果存在两核之间物理地址不一致的情况，必须实现此接口。

资源表结构
========================

RPTUN 使用的标准资源表结构：

.. code-block:: c

   struct aligned_data(8) rptun_rsc_s
   {
     struct resource_table    rsc_tbl_hdr;
     uint32_t                 offset[2];
     struct fw_rsc_trace      log_trace;
     struct fw_rsc_vdev       rpmsg_vdev;
     struct fw_rsc_vdev_vring rpmsg_vring0;
     struct fw_rsc_vdev_vring rpmsg_vring1;
     struct fw_rsc_config     config;
   };

有关资源表的详细信息，请参阅 OpenAMP 文档。
