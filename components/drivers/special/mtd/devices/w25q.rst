==============
W25Q NOR Flash
==============

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

NuttX 为 W25Q 系列的 QSPI 和 SPI NOR Flash 提供支持。
基于 QSPI 的 Flash 支持最高 1 Gbit（128 MB）容量，
基于 SPI 的 Flash 支持最高 128 Mbit（16 MB）容量。

----------
QSPI flash
----------

基于 QSPI 的 Flash 通过选项 ``CONFIG_MTD_W25QXXXJV`` 启用。
可以通过 ``CONFIG_W25QXXXJV_QSPIMODE`` 选项选择 QSPI 模式，
并通过 ``CONFIG_W25QXXXJV_QSPI_FREQUENCY`` 选项选择通信频率。

如果启用了 ``CONFIG_W25QXXXJV_SECTOR512`` 选项，该 Flash 允许
模拟 512 字节的大擦除块。

Flash 的大小在设备初始化期间自动设置。

闪存在使用前必须初始化。这通常在板卡启动阶段从板级支持包层完成。
此操作由以下函数执行。

.. code-block:: C

   #include <nuttx/mtd/mtd.h>

   FAR struct mtd_dev_s *w25qxxxjv_initialize(FAR struct qspi_dev_s *qspi,
                                              bool unprotect)

---------
SPI flash
---------

基于 SPI 的 Flash 通过选项 ``CONFIG_MTD_W25`` 启用。可以通过
``CONFIG_W25_SPIMODE`` 选项选择 SPI 模式，并通过
``CONFIG_W25_SPIFREQUENCY`` 选项选择通信频率。另一个重要的配置
是 SPI 延迟的设置。与 QSPI 内存模式下所有延迟都设置为 0 不同，
SPI 外设可能有不同的默认延迟。W25 Flash 的延迟可以通过
``CONFIG_W25_START_DELAY``、``CONFIG_W25_STOP_DELAY``、
``CONFIG_W25_CS_DELAY`` 和 ``CONFIG_W25_IFDELAY`` 选项设置。
默认值设置为零，因此传输和片选之间没有延迟。

如果启用了 ``CONFIG_W25_SECTOR512`` 选项，该 Flash 允许模拟
512 字节的大擦除块。如果设置了 ``CONFIG_W25_READONLY``，还可以
将 Flash 配置为只读。

Flash 的大小在设备初始化期间自动设置。

闪存在使用前必须初始化。这通常在板卡启动阶段从板级支持包层完成。
此操作由以下函数执行。

.. code-block:: C

   #include <nuttx/mtd/mtd.h>

   FAR struct mtd_dev_s *w25_initialize(FAR struct spi_dev_s *spi)
