=======================
键盘/按键驱动
=======================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/


**按键与键盘**。对于 NuttX 来说，键盘和按键实际上是相同的设备。按键被认为是键数较少的简单键盘。

**特殊命令**。在 NuttX 中，键盘/按键驱动程序只是一个字符驱动程序，可能在字符驱动程序返回的数据上有一个（可选的）编码/解码层。当按下键时，键盘可以返回简单的文本数据（字母、数字和标点符号）或控制字符（回车、control-C 等）。我们可以将此视为"正常"键盘数据流。然而，此外，大多数键盘支持无法表示为文本或控制数据的操作。这些操作包括光标控制（Home、上箭头、Page Down 等）、编辑功能（插入、删除等）、音量控制（静音、音量增大等）和其他特殊功能。在这种情况下，可能需要一些特殊编码来多路复用正常的文本数据和特殊命令键按键数据流。

**按键和释放事件**。有时应用程序也需要知道键释放的时间。因此，除了正常和特殊按键事件外，可能还需要编码正常和特殊键释放事件。

**编码/解码层**。可选的编码/解码层可与基本字符驱动程序一起使用，将键盘事件编码到文本数据流中。构成该编码/解码层的函数接口在头文件 ``include/nuttx/input/kbd_code.h`` 中定义。这些函数提供了一组匹配的（a）驱动程序编码接口和（b）应用程序解码接口。

#. **应用程序解码接口**。这些是用于解码键盘/按键驱动程序返回的值的用户接口。

   -  ``kbd_decode()``

      **函数原型：**

      **描述：**

      **输入参数：**

      -  ``stream``：用于执行实际底层 get 操作的 ``lib_instream_s`` 实例。
      -  ``pch``：保存返回值的位置。这可以是正常的字符代码或特殊命令（即来自 ``enum kbd_getstate_s`` 的值）。
      -  ``state``：用户提供的用于支持解析的缓冲区。首次调用 ``kbd_decode()`` 时应清除此结构。

      **返回值：**

      -  ``KBD_PRESS`` (0)**：表示成功接收到正常的键盘数据。这对应于按键事件。``pch`` 中的返回值是简单的文本或控制数据字节。
      -  ``KBD_RELEASE`` (1)**：表示键释放事件。``pch`` 中的返回值是与释放的键对应的文本或控制数据字节。
      -  ``KBD_SPECPRESS`` (2)**：表示成功接收到特殊键盘命令。``pch`` 中的返回值是来自 ``enum kbd_getstate_s`` 的值。
      -  ``KBD_SPECREL`` (3)**：表示特殊命令键释放事件。``pch`` 中的返回值是来自 ``enum kbd_getstate_s`` 的值。
      -  ``KBD_ERROR`` (``EOF``)**：获取下一个字符时出错（由 ``stream`` 报告）。通常表示文件结束。
