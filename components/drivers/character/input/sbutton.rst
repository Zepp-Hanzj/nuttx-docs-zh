==================================
单按钮多功能驱动程序
==================================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

**单按钮（SButton）** 是一种键盘驱动，仅使用板上的单个物理按钮（开关）。这种按钮用于简单界面，例如 3D 打印机或其他只需移动到下一个选项并确认选择的设备。

它通过检测按钮是短按（即少于 500ms）还是长按来实现。如果是短按，驱动程序将返回 **TAB**；如果是长按，驱动程序将返回 **ENTER**。使用这种方式可以在类似菜单中进行导航。

**工作原理**。驱动程序使用简单的配置数据（此配置数据等同于 Linux 内核中的平台数据）来映射用于注册和检测中断的 MCU 引脚，该引脚物理连接到按钮。

它使用一种 C 语言中的"多态性"来允许驱动程序访问负责附加和启用中断的函数以及获取引脚状态。
参见 ``include/nuttx/input/sbutton.h``
和 ``boards/arm/stm32/common/src/stm32_sbutton.c`` 以更好地理解其工作原理。基本上，板级文件（配置数据）创建一个结构体，其第一个字段（变量）是 SButton 驱动程序（``drivers/input/sbutton.c``）使用的配置结构体。

每次用户按下或释放按键时都会产生一个中断。sbutton 中的中断 ISR（``sbutton_interrupt()``）调用一个工作队列来处理它（因为我们不能在 ISR 中花费时间处理数据，这会降低 RTOS 的性能）。该工作队列（``sbutton_worker()``）需要测量从按键按下到释放之间的经过时间（tick），以决定它是 "KEY_1"（**TAB**）还是 "KEY_2"（**ENTER**）。然后调用键盘上层的 ``keyboard_event()`` 将此按键发送给用户应用程序。
