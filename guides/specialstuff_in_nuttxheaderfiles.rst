=======================================================
为什么不能把我的特殊内容放在 NuttX 头文件中？
=======================================================

.. warning::
    迁移自：https://cwiki.apache.org/confluence/pages/viewpage.action?pageId=139629466

问题
===========

我对 NuttX 头文件中包含的内容非常挑剔。我不接受非标准化的更改仅为了让外部代码能够编译而放入它们；也不接受暗示 NuttX 中实现了某些功能但实际上并未实现的更改（尽管头文件中现在确实存在这样的情况）。

* 但我想要使用依赖于 NuttX 头文件中某些定义的 `newlib` 逻辑！
* 但我正在尝试编译一个依赖于头文件中非标准声明的应用程序！或者 NuttX 未提供的函数原型！或者 NuttX 未使用的类型！

你会发现我在这个问题上非常固执，如果你试图为了个人目的将无用内容包含到 NuttX 头文件中，你将浪费你的时间和精力。

变通方法
=============

但是有一个变通方法可以解决我的挑剔和固执（至少对于支持 GNU 扩展的编译器如 GCC）。假设你想将以下定义添加到标准 ``time.h`` 头文件中：

.. code-block:: c

    #define I_AM_A_NERD true

你提交了一个补丁来做到这一点，我拒绝了。现在怎么办？

虽然我拒绝将非标准或无用的内容放入 NuttX 头文件中，但有办法解决这个问题。假设你创建了一个名为 ``myincludes/`` 的目录，在你的 ``myincludes/`` 目录中有一个名为 ``time.h`` 的头文件。这个 ``time.h`` 头文件包含：

.. code-block:: C

    #define I_AM_A_NERD true
    #include_next <time.h>

然后在你的 ``CFLAGS`` 中，你使用 ``-isystem`` 设置在 NuttX ``include/`` 目录之前的 ``myincludes/`` 中包含头文件。当你的应用程序包含 ``time.h`` 时，``myincludes/`` 中的 ``time.h`` 版本将被包含。该版本将按你的意愿定义 ``I_AM_A_NERD``，然后包含编译器包含路径中下一个名为 ``time.h`` 的文件。该文件将是 NuttX ``include/`` 目录中提供的标准 ``time.h`` 头文件。

通过这种方式，你可以附加或修改任何 NuttX 头文件以满足你自己的目的，而我不必将我不想接受的更改放入 NuttX 仓库。

什么时候有意义？
========================

什么时候向 NuttX 头文件添加新的定义、类型和函数原型才有意义？只有在以下条件下：

* 更改是标准的，并在 OpenGroup.org 中指定
* 更改由包含使用类型和实现函数的完整、经过验证的功能实现的补丁提供。
