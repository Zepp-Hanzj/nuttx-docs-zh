``bitmap_converter.py`` NxWidgets
=================================

此脚本将 Python 成像库支持的任何图像类型转换为 NxWidgets 使用的 RLE 编码格式。

RLE（游程编码）是一种非常简单的编码方式，对某些类型的图像具有良好的压缩效果：
在一行中有许多相同颜色像素相邻的图像（如简单图形）。它对摄影图像效果不佳。

但即使是简单图形，如果经过调整大小，也可能无法紧凑编码。调整图像大小可能会创建
数百种唯一颜色，这些颜色在 RGB 表示中可能仅相差一两位。这种"颜色涂抹"是像素
插值的结果（如果您的图形软件支持通过像素复制而非插值来调整大小，则可以消除此问题）。

当简单图形图像编码效果不佳时，症状是生成的 RLE 数据结构相当大。特别是调色板结构
可能包含数百种颜色。在这种情况下，有一种方法可以修复图形图像。具体操作如下：

- 在 GIMP 中打开原始图像。
- 选择图像颜色数量选项。
- 选择能忠实表示图像的最少颜色数。对于大多数简单图形图像，可能只需 6 或 8 种颜色。
- 将图像保存为 PNG 或其他无损格式（不要使用 jpeg）。
- 然后生成图像。

``mkromfsimg.sh``
=================

**问**：为什么有两个版本的 ``mkromfsimg.sh`` 脚本，一个在 ``apps/tools`` 中，
另一个在 ``nuttx/tools`` 中。

**答**：``nuttx/tools`` 中的 ``mkromfsimg.sh`` 版本是一个通用工具，用于简化
从任何目录创建 ROMFS 文件系统，该目录包含您希望在目标系统中访问的内容。

而 ``apps/tools`` 中的版本则有非常特殊的用途。它是可在 KERNEL 构建模式下使用的
支持的一部分。

内核构建中的进程和程序
------------------------------------------

在内核构建中，没有任务（task）。只有进程（process），所有代码都在自己的私有地址
空间中运行。参见 :doc:`/implementation/processes_vs_tasks`。

这导致的一个结果是，像 ``task_create()`` 这样的函数及其相关函数不能在 KERNEL
构建模式下使用。相反，所有进程必须从文件系统中的 ELF 或 NxFLAT 文件加载到虚拟
地址空间中。ROMFS 是众多文件系统之一，但在深度嵌入式系统中特别适合此用途。

内核构建的差异
------------------------

在 FLAT 和 PROTECTED 构建模式中，所有应用程序构建为单个 BLOB，因此每个符号
必须具有唯一名称以避免名称冲突。

在 KERNEL 构建模式中，所有应用程序构建为独立链接的程序，驻留在文件系统中。
所有程序的入口点都是 ``main()`` 函数。

apps/bin
--------

在 FLAT 或 PROTECTED 模式下构建 ``apps/`` 程序时，所有目标文件被放入存档
apps/libapps.a 中，最终复制到 ``nuttx/libs``，BLOB 通过链接 NuttX 存档和
``lib/libapps.a`` 创建。

但在 KERNEL 模式下构建 ``apps/`` 程序时，``apps/bin`` 目录由顶层 apps/Makefile
创建。每个源文件都被编译，但目标文件不会添加到任何存档中。相反，目标文件被链接为
独立的编译和链接程序。每个程序然后安装到 ``apps/bin`` 中。

apps/tools/mkromfsimg.sh
------------------------

当 ``apps/`` 内核构建完成时，所有程序都已安装在 ``apps/bin`` 中。这就是
``apps/tools/mkromfsimg.sh`` 文件发挥作用的地方。它获取 apps/bin 中的所有程序，
并创建包含所有应用程序的 ROMFS 文件系统映像。该 ROMFS 文件系统映像被构建到内核中。

应用程序初始化
--------------------------

在运行时，当内核启动时，它会将该 ROMFS 文件系统挂载到 ``/bin``。在 FLAT 构建模式中，
OS 启动逻辑调用 ``task_create()`` 来启动您使用 ``CONFIG_INIT_ENTRYPOINT`` 配置的
初始任务。但在 KERNEL 构建中，会发生不同的事情。不使用 ``CONFIG_INIT_ENTRYPOINT``。
而是使用 ``CONFIG_INIT_FILEPATH``。这将是 ``/bin`` 中要启动以启动系统的程序名称。
