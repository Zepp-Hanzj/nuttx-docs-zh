=================
MDIO 总线驱动
=================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

NuttX MDIO 总线驱动提供了一个标准化接口，用于与以太网 PHY（物理层）收发器通信。
它采用经典的上层/下层架构，将硬件特定逻辑与通用 MDIO 协议抽象分离，
目前符合 IEEE 802.3 标准的第 22 条。
上层的主要实现可以在 ``drivers/net/mdio.c`` 中找到。

驱动架构
===================

MDIO 驱动框架作为网络设备驱动和物理总线之间的中间层。
预期的操作模型是 ``netdev -> phydev -> mdio``，即网络设备与专用 PHY 驱动通信，
PHY 驱动再使用 MDIO 总线驱动进行低层硬件访问。
不建议网络设备与 MDIO 总线之间直接交互。

上层实现
-------------------------

上层驱动包含 MDIO 总线的核心逻辑，包括确保安全事务的总线锁定机制。
它公开了一个通用 API 来管理总线生命周期，并能够并发处理多个独立的 MDIO 总线实例。
这为 PHY 驱动和底层硬件特定代码抽象了实现细节。

下层实现
-------------------------

下层 MDIO 驱动是一个薄层，将上层定义的通用操作映射到硬件特定的寄存器操作。
它不旨在包含复杂逻辑，而是为总线操作提供直接的翻译。

实现下层驱动
================================

为新硬件集成 MDIO 支持需要实现下层驱动。
上下层之间的契约定义在 ``include/nuttx/net/mdio.h`` 中，
围绕两个关键结构展开。

关键数据结构
-------------------

1.  ``struct mdio_ops_s``：一个包含函数指针的结构，下层驱动必须实现
    这些函数以执行硬件级操作。
    *   ``read``：执行第 22 条 MDIO 读操作。
    *   ``write``：执行第 22 条 MDIO 写操作。
    *   ``reset``：一个可选函数，用于执行硬件特定的 PHY 复位。

2.  ``struct mdio_lowerhalf_s``：下层实例的容器，持有指向 ``mdio_ops_s``
    虚函数表的指针以及用于驱动内部状态的可选私有数据指针。

注册和注销
-------------------------------

板级初始化逻辑负责实例化下层驱动并通过 ``mdio_register()`` 函数
将其注册到上层。使用不同的下层驱动调用此函数会创建新的唯一总线句柄，
允许系统并发管理多个 MDIO 总线。

.. code-block:: c

    FAR struct mdio_dev_s *mdio_register(FAR struct mdio_lowerhalf_s *lower);

此函数接受下层实例并返回一个不透明句柄（``FAR struct mdio_dev_s *``），
PHY 驱动随后使用该句柄与总线交互。

当不再需要总线实例时，应通过调用 ``mdio_unregister()`` 函数
来释放它，以确保资源的正确清理。

.. code-block:: c

    int mdio_unregister(FAR struct mdio_dev_s *dev);

此函数接受 ``mdio_register()`` 返回的句柄并释放关联的总线实例。

一个（基本）完整的下层驱动参考实现可在 ``arch/arm/src/stm32h7/stm32_mdio.c`` 中找到。
