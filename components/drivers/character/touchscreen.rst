==========================
触摸屏设备驱动程序
==========================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

NuttX 支持两部分触摸屏驱动程序架构。

#. "上半部分"，通用驱动程序，为应用程序代码提供通用的触摸屏接口，
#. "下半部分"，平台特定的驱动程序，实现底层触摸屏控制以实现触摸屏功能。

支持触摸屏控制器（TSC）驱动程序的文件可以在以下位置找到：

-  **接口定义**。NuttX 触摸屏驱动程序的头文件位于 ``include/nuttx/include/input`` 目录。触摸屏控制器"上半部分"和"下半部分"驱动程序之间的接口*不是*通用的，而是因控制器而异。因此，每个触摸屏驱动程序在该目录中有自己独特的头文件来描述"上半部分"/"下半部分"接口。另一方面，每个触摸屏驱动程序的应用程序级别接口*是*相同的，描述在 ``include/nuttx/include/input/touchscreen.h``。触摸屏驱动程序使用标准字符驱动程序框架，但读操作返回特殊格式的数据。
-  **"上半部分"驱动程序**。控制器特定的"上半部分"触摸屏驱动程序位于 ``drivers/input`` 目录。
-  **"下半部分"驱动程序**。平台特定的触摸屏驱动程序位于以下位置之一：(1) 对于具有内置触摸屏控制器的处理器架构，位于 ``arch/<architecture>/src/<hardware>`` 目录，或 (2) 对于使用外部触摸屏控制器芯片的板，位于 ``boards/<arch>/<chip>/<board>/src/`` 目录。

应用程序编程接口
=================================

要在应用程序中使用触摸屏驱动程序，首先需要包含正确的头文件。它包含驱动程序的应用程序编程接口。为此，包含

.. code-block:: c

  #include <nuttx/input/touchscreen.h>

触摸屏驱动程序注册为 ``/dev`` 命名空间中的 POSIX 字符设备文件。需要打开设备以获取文件描述符以进行后续操作。这可以通过标准的 POSIX ``open()`` 调用完成。

驱动程序通过 ``read``、``write``、``poll`` 和 ``ioctl`` 接口访问。以下 ``ioctl`` 命令可用：

 * :c:macro:`TSIOC_GRAB`

.. c:macro:: TSIOC_GRAB

此命令让当前句柄抓取设备。当句柄抓取设备时，它成为来自该设备的所有触摸屏事件的唯一接收者。参数是一个 ``int32_t`` 变量，用于启用或禁用抓取。
