===========================================
``bare`` 二进制应用记录编码
===========================================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

BARE 是一种用于结构化应用数据的简单二进制表示格式，请参阅
https://baremessages.org/。

移植到 NuttX 的 BARE 实现是 ``cbare``，请参阅
https://git.sr.ht/~fsx/cbare。

BARE_ 类似于其他二进制格式，如 CBOR_、BSON_ 或 `Protocol
Buffers`_。

.. _BARE: https://baremessages.org/
.. _CBOR: https://cbor.io/
.. _BSON: https://bsonspec.org/
.. _Protocol Buffers: https://protobuf.dev/

BARE 概览，摘自其网页：

- 消息以二进制编码，体积紧凑。消息不包含模式信息——它们不是自描述的。

- BARE 针对小消息进行了优化。它没有针对在单条消息中编码大量数据或高效读取具有固定大小字段的消息进行优化。但是，所有类型都对齐到 8 位，这确实以空间换取了简洁性。

- BARE 的可扩展性方法是保守的：今天编码的消息明天可以解码，反之亦然。但可扩展性仍然是可能的；实现可以选择在更高层次上解码用户定义的类型，并将它们映射到任意数据类型。

- 规范同样是保守的。消息解码器和编码器的简单实现可以在一个下午内编写完成。

- 提供了一个可选的 DSL 来记录消息模式并为代码生成提供来源。但是，如果您愿意，也可以使用编程语言中已有的类型系统来定义您的模式。
