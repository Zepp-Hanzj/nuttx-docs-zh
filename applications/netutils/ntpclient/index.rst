========================
``ntpclient`` NTP 客户端
========================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

NTP（网络时间协议）客户端是一个网络工具，用于将系统时钟与 Internet 上的时间服务器同步。此实现为 NuttX 提供了一个最小但功能完整的 NTP 客户端。

什么是 NTP？
============

网络时间协议（NTP）是一种网络协议，旨在通过分组交换、可变延迟的数据网络同步计算机系统的时钟。NTP 是目前仍在使用的最古老的 Internet 协议之一，最初由特拉华大学的 David L. Mills 设计。

NTP 的主要特性：

- **高精度**：NTP 在局域网上可以实现亚毫秒级精度，在 Internet 上通常可以实现 10-100 毫秒精度
- **鲁棒算法**：使用复杂的算法过滤网络抖动并选择最佳时间源
- **层次结构**：使用层级系统，其中层级 0 设备是原子钟，层级 1 服务器与层级 0 同步，依此类推
- **容错性**：可以处理多个时间源并在它们之间自动切换

NTP 协议概述
=====================

NTP 使用 UDP 端口 123，遵循客户端-服务器模型。协议交换时间戳以计算：

- **偏移量**：客户端时钟与服务器时钟之间的差异
- **延迟**：往返网络延迟
- **离散度**：由于时钟频率容差造成的最大误差

NTP 数据包格式（版本 3）包括：

- **闰秒指示器**：警告即将到来的闰秒
- **版本号**：NTP 版本（此实现中为 3）
- **模式**：客户端、服务器、广播等
- **层级**：时钟级别（0-15）
- **轮询间隔**：连续消息之间的最大间隔
- **精度**：时钟精度
- **根延迟/离散度**：到参考时钟的总延迟和离散度
- **参考标识符**：标识参考源
- **参考时间戳**：系统时钟上次设置的时间
- **发起时间戳**：请求离开客户端的时间
- **接收时间戳**：请求到达服务器的时间
- **发送时间戳**：应答离开服务器的时间

实现细节
======================

NuttX NTP 客户端实现由几个关键组件组成：

源代码结构
---------------------

**ntpclient.c** - 主实现文件，包含：

- **守护进程管理**：启动、停止和管理 NTP 守护进程的函数
- **时间同步**：计算时钟偏移和延迟的核心算法
- **网络通信**：UDP 套接字处理和 NTP 数据包交换
- **样本收集**：收集多个时间样本以进行统计过滤

**ntpv3.h** - NTP 版本 3 数据包格式定义：

- **ntp_datagram_s**：完整的 NTP 数据包结构
- **ntp_timestamp_s**：64 位 NTP 时间戳格式
- **协议常量**：NTP 版本、模式和层级定义

关键函数
-------------

- **ntpc_start_with_list()**：使用服务器列表启动 NTP 守护进程
- **ntpc_start()**：使用默认配置启动 NTP 守护进程
- **ntpc_stop()**：停止正在运行的 NTP 守护进程
- **ntpc_status()**：获取当前同步状态和样本
- **ntpc_daemon()**：主守护进程循环：

  - 连接到配置的 NTP 服务器
  - 发送 NTP 请求并处理响应
  - 计算时钟偏移和延迟
  - 将时间校正应用于系统时钟
  - 继续按配置的间隔进行轮询

- **ntpc_get_ntp_sample()**：执行单次 NTP 事务：

  - 创建到 NTP 服务器的 UDP 套接字
  - 发送带有当前时间戳的 NTP 请求数据包
  - 接收并验证 NTP 响应
  - 使用 NTP 算法计算偏移和延迟

- **ntpc_calculate_offset()**：实现 NTP 时钟过滤器算法

  - 使用四个时间戳，计算偏移和延迟
  - 应用统计过滤以减少抖动

- **ntpc_settime()**：将时间校正应用于系统时钟：

  - 使用计算的偏移调整系统时间
  - 处理正向和负向时间调整
  - 保持单调时钟一致性

配置选项
=====================

NTP 客户端可以通过 Kconfig 选项进行配置：

- **CONFIG_NETUTILS_NTPCLIENT_SERVER**：NTP 服务器主机名列表
- **CONFIG_NETUTILS_NTPCLIENT_PORTNO**：NTP 服务器端口（默认：123）
- **CONFIG_NETUTILS_NTPCLIENT_STACKSIZE**：守护进程任务堆栈大小
- **CONFIG_NETUTILS_NTPCLIENT_SERVERPRIO**：守护进程任务优先级
- **CONFIG_NETUTILS_NTPCLIENT_STAY_ON**：保持持续轮询
- **CONFIG_NETUTILS_NTPCLIENT_POLLDELAYSEC**：轮询间隔（秒）
- **CONFIG_NETUTILS_NTPCLIENT_NUM_SAMPLES**：用于过滤的样本数
- **CONFIG_NETUTILS_NTPCLIENT_TIMEOUT_MS**：网络超时（毫秒）

用法
=====

NTP 客户端通常通过系统命令使用：

.. note:: NTP 客户端功能需要在配置中启用 :code:`SYSTEM_NTPC` 选项。请确保在构建前在 menuconfig 或 Kconfig 片段中选择此选项。

- **ntpcstart**：启动 NTP 守护进程
- **ntpcstop**：停止 NTP 守护进程
- **ntpcstatus**：显示同步状态

示例工作流程：

1. 配置网络连接
2. 启动 NTP 客户端：``ntpcstart``
3. 检查状态：``ntpcstatus``
4. 验证时间：``date`` 命令
5. 需要时停止：``ntpcstop``

客户端将自动：
- 连接到配置的 NTP 服务器
- 交换时间信息
- 计算并应用时钟校正
- 继续定期同步

限制
===========

这是一个最小的 NTP 客户端实现，有一些限制：

- **无认证**：不支持 NTP 认证（MD5/SHA1）
- **基本过滤**：使用简单的统计过滤，不是完整的 NTP 算法
- **单一参考**：不实现完整的 NTP 参考时钟选择
- **无闰秒处理**：不处理闰秒公告
- **有限的错误处理**：基本的错误恢复和重试机制

尽管有这些限制，该实现为大多数需要网络时间同步的嵌入式应用提供了足够的精度。

依赖
============

NTP 客户端需要：

- **CONFIG_NET**：网络支持
- **CONFIG_NET_UDP**：UDP 协议支持
- **CONFIG_NET_SOCKOPTS**：套接字选项支持
- **CONFIG_LIBC_NETDB**：DNS 解析（推荐）

为了获得最佳效果，请确保：
- 稳定的网络连接
- 访问可靠的 NTP 服务器
- 足够的系统资源用于守护进程运行
