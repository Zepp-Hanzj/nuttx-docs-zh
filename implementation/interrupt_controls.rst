=============================
每线程中断控制
=============================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

使用 NuttX 时，您会发现中断启用/禁用状态不是全局属性。您不能只是为所有任务关闭和打开中断。相反，启用和禁用中断只在控制中断的单个任务运行时生效。考虑以下序列：

.. code-block:: C

   irqstate_t flags;
 
   flags = irqsave();   /* 禁用中断 */
   sleep(5);            /* 睡眠 5 秒 */
   irqrestore(flags);   /* 重新启用中断 */

当任务睡眠时会发生什么？这是否意味着中断将被禁用五秒钟？不，中断可能会在任务睡眠期间被重新启用。这是如何工作的？

实际上非常简单。每次发生上下文切换时，都会为被挂起的任务保存一组寄存器。然后从先前保存的寄存器中恢复这些寄存器，用于将要运行的下一个任务。这就是为什么我们经常将上下文切换描述为只是增强版的 setjmp/longjmp：上下文切换的工作方式就像 setjmp（保存一组寄存器）和 longjmp（恢复一组寄存器），只是保存和恢复的寄存器更多。


以 ARMv7-M 为例，您可以在 ``arch/arm/include/armv7-m/irq.h`` 中看到存储的寄存器集

在保存和恢复的寄存器中，有决定中断是否启用的寄存器。对于 ARMv7-M 系列，这是 ``PRIMASK`` 寄存器或 ``BASEPRI`` 寄存器。因此，如果一个任务禁用中断然后挂起，``PRIMASK``/``BASEPRI`` 寄存器的当前值会被保存，并被下一个将要运行的任务的 ``PRIMASK``/``BASEPRI`` 寄存器的存储值替换，从而在第一个任务挂起时重新启用中断。

因此，中断启用/禁用是每线程属性，而不是全局属性。如果您长期从事裸机系统开发，这可能会让您感到陌生。

顺便说一下，通过 ``sched_lock()`` 锁定调度器的行为方式与此相同（但机制略有不同）。
