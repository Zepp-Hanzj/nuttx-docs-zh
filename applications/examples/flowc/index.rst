======================================
``flowc`` 串口硬件流控
======================================

一个简单的串口硬件流控测试。

一般使用说明：

1. 在接收端，启动接收程序。接收端现在等待在配置的串口上接收数据。
2. 在发送端，启动发送程序。该程序将向接收端发送数据，接收端会验证数据是否丢失。

在 Linux 上，也可以使用以下方式：

  $ stty -F /dev/ttyACM0 crtscts
  $ cat testdata.dat >/dev/ttyACM0

其中需要将 ``/dev/ttyACM0`` 替换为您选择的串口设备。
