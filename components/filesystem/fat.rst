===
FAT
===

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

文件分配表（File Allocation Table, FAT）是由 Microsoft 设计的一个非常简单的文件系统，顾名思义，它使用一个表来跟踪存储卷上的簇。FAT 有多个版本，如 ``FAT12``、``FAT16``，以及最后的 ``FAT32``，以应对不断增长的数据容量。尽管在大多数 Windows 计算机中 FAT 已经被更新的文件系统所取代，但当前的 Windows 版本、macOS、Linux、Unix 以及 Apache NuttX 都支持 FAT。它仍然用于软盘、USB 闪存驱动器以及数码相机等小型便携设备中的存储介质。

FAT 使用魔术数字 ``0x4D44``，在代码库中被命名为 ``vfat``，因为实际实现的是 VFAT，它是允许长文件名的扩展。本页面包含从 NuttX 中 FAT 实现的角度出发的 FAT 文件系统信息。

FAT 结构
=============

``FAT`` 旁边的数字（例如 ``FAT16`` 中的 ``16``）表示分配表中每个条目使用的位数。FAT 卷有多个部分：

* **主引导记录（Master Boot Record）**: 主引导记录（MBR）或引导扇区包含文件系统访问卷所需的信息，包括卷的布局和文件系统结构，类似于 Linux 文件系统中的超级块。
* **分配表区域**: 存储文件分配表及其备份副本。文件分配表跟踪卷中的所有簇，因此每个簇都有一个条目。
* **根目录**: 存储一个目录表，其中包含描述卷上存储的文件和目录的条目。每个条目包含有关文件系统对象的元数据。

.. NOTE::
  如果卷是 ``FAT12`` 或 ``FAT16``，根目录作为单独的部分存在于分配表区域之后，并且对条目数量有固定的上限。``FAT32`` 没有专用的根目录区域，而是将其纳入下面的数据区域中，因此 ``FAT32`` 对条目数量没有任何硬编码上限。

* **数据区域**: 此区域存储文件的实际数据以及目录数据。数据区域分为多个簇，这些簇按顺序编号并在分配表中有相应的条目。簇大小可能因卷大小和 FAT 文件系统类型而异。

分配表中的第一个条目用于卷的 FAT ID，而第二个条目表示该簇是保留的。鉴于 ``FAT32`` 没有专用的根目录部分，在 ``FAT32`` 卷中，第三个条目指向根目录。

文件名可以是 `短文件名 <https://en.wikipedia.org/wiki/8.3_filename>`_（也称为 8.3 文件名或 SFN）或长文件名（LFN），直到最大长度限制。

主引导记录（MBR）
========================

引导记录可能是两种类型之一。一种是较旧的类型，不包含分区，另一种是较新的类型，包含分区。FAT 的一个显著特征是所有 MBR 在偏移量 510 处都包含 ``0x55AA``，这可用于确定块设备中使用的 FAT 类型。

对于较旧的类型，MBR 位于驱动器的起始位置。它不包含任何分区。

较新的类型在驱动器起始偏移量 446 处有一个分区表。它允许 4 个主分区。在每个分区的起始位置都有 FAT 引导记录（FBR），其结构在大多数方面与较旧的 MBR 相同。

分区表条目
=======================

分区表包含 4 个主分区的信息，每个分区在表中有 16 字节的条目。这些条目包含有关分区的各种信息，包括但不限于分区类型、起始扇区和分区大小。

分配表
================

分配表跟在引导记录之后，对数据区域中每个可用的簇都有一个条目。每个条目的大小由文件系统类型定义。``FAT12`` 的条目长度为 12 字节，``FAT16`` 为 16 字节，``FAT32`` 为 32 字节。

文件数据由簇的链表（或链）组成。分配表中对应于实际簇的条目包含该文件中下一个簇的簇号。文件的最后一个簇在其分配表条目中有一个所有位都被设置的值（FAT12 为 ``0xFFF``，FAT16 为 ``0xFFFF``，FAT32 为 ``0xFFFFFFFF``）。

起始簇（文件链的头部）由该文件的目录条目指向。

目录条目
=================

目录基本上是一个文件，其中有一个表包含目录条目，这些条目包含其中文件和子目录的元数据。每个目录条目描述目录中的一个文件或子目录。

FAT12/16 中的根目录有一个专用区域，与数据区域分开。在 FAT32 中，根目录就像数据区域中的普通目录一样（即没有专用区域），除了：

* 其起始簇在引导记录中标明。
* 它不被任何其他目录指向。
* 它没有对应于 ``.`` 和 ``..`` 的条目（所有非根目录都有这两个条目）。

目录条目长度为 32 字节，以 11 字节的 SFN 开头。目录条目的其余部分包含文件属性、创建时间戳以及访问和写入时间戳、簇号和文件大小等信息。

8.3 文件名
------------

目录的 8.3 文件名（因此也是整个目录条目）的第一个字节有特殊含义。它有 3 个可能的值：

* ``0xe5`` : 表示当前目录条目为空。
* ``0x00`` : 表示此条目以及所有后续条目均为空。
* ``0x05`` : 实际值为 ``0xe5``。

目录条目剩余的 7 + 3 字节用于名称（含扩展名）。

文件
=====

文件中数据的起始簇由文件的目录条目指向。

FAT 中的文件分配非常简单，卷中的第一个可用位置会被分配给文件。

文件属性
--------------------

文件属性由单字节大小的位标志表示。FAT 中的文件标志及其位表示如下：

.. list-table:: 文件属性
  :header-rows: 1

  * - 属性宏
    - 位表示
    - 十六进制值
  * - ``FATATTR_READONLY``
    - ``00000001``
    - 0x1
  * - ``FATATTR_HIDDEN``
    - ``00000010``
    - 0x2
  * - ``FATATTR_SYSTEM``
    - ``00000100``
    - 0x4
  * - ``FATATTR_VOLUMEID``
    - ``00001000``
    - 0x8
  * - ``FATATTR_DIRECTORY``
    - ``00010000``
    - 0x10
  * - ``FATATTR_ARCHIVE``
    - ``00100000``
    - 0x20

其中，FAT 向用户暴露 ``FATATTR_READONLY``、``FATATTR_HIDDEN``、``FATATTR_SYSTEM`` 和 ``FATATTR_ARCHIVE``。

==============
