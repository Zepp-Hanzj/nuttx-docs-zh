====================================
ARM64 MTE 扩展
====================================

.. note:: 本文档翻译自 NuttX 官方文档，如需查阅最新版本请访问 https://nuttx.apache.org/docs/latest/

简介
------------

Arm v8.5 引入了 Arm 内存标签扩展（MTE），这是标记内存的硬件实现。

基本上，MTE 为每个内存分配/释放添加额外的元数据标签。它为内存位置分配一个标签，然后可以与引用该内存位置的指针关联。在运行时，CPU 在每次加载和存储时检查指针和元数据标签是否匹配。

NX OS 目前支持在 ARM64 QEMU 上部署 MTE，这在 NX OS 的 EL1 级别受支持。

原理
---------

Arm 内存标签扩展实现了锁和钥匙的内存访问。可以在内存上设置锁，并在内存访问时提供钥匙。如果钥匙与锁匹配，则允许访问。如果不匹配，则报告错误。

通过向每 16 字节物理内存添加 4 位元数据来标记内存位置。这就是标签粒度（Tag Granule）。标记内存实现了锁。指针（因此虚拟地址）被修改以包含钥匙。为了在不需要更大指针的情况下实现钥匙位，MTE 使用了 Armv8-A 架构的 Top Byte Ignore（TBI）功能。当 TBI 启用时，虚拟地址的高位字节在用作地址转换输入时被忽略。这允许高位字节存储元数据。在 MTE 中，高位字节的 4 位用于提供钥匙。

架构详情
---------------------

MTE 向 Armv8-A 架构添加了以下指令，分为三个不同类别 [6]：
适用于栈和堆标签的标签操作指令。

IRG
为了使 MTE 的统计基础有效，需要随机标签的来源。IRG 被定义为在硬件中提供此功能，并将此类标签插入寄存器中供其他指令使用。

GMI
此指令用于操作与 IRG 指令一起使用的排除标签集。这适用于软件将特定标签值用于特殊目的，同时保留正常分配的随机标签行为的情况。

LDG、STG 和 STZG
这些指令允许在内存中获取或设置标签。它们旨在更改内存中的标签，而不修改数据或将数据清零。

ST2G 和 STZ2G
这些是 STG 和 STZG 的更密集替代方案，在分配大小允许时操作两个粒度的内存。

STGP
此指令将标签和数据存储到内存中。
用于指针算术和栈标签的指令：

ADDG 和 SUBG
这些是 ADD 和 SUB 指令的变体，用于地址算术。它们允许标签和地址分别由立即值修改。这些指令旨在创建栈上对象的地址。

SUBP(S)
此指令提供 56 位减法（可选标志设置），这是忽略高位字节中标签的指针算术所必需的。

用于系统的指令：
LDGM、STGM 和 STZGM
这些是批量标签操作指令，在 EL0 未定义。这些旨在供系统软件操作标签以进行初始化和序列化。例如，它们可用于实现将标记内存交换到非标签感知介质。清零形式可用于高效初始化内存。

当前 NX OS 支持执行上述指令，如 irg、ldg、stg 指令。它们的测试程序存储在 "apps/system/mte" 中，用于测试当前系统是否支持。

用法
-----

如果你想体验 NX OS 的 MTE 功能，可以参考以下内容：
要启用 ARM64_MTE，请按如下配置内核::

    CONFIG_ARM64_MTE=y

当然，你也可以使用现有配置运行：

    boards/arm64/qemu/qemu-armv8a/configs/mte
