===========================================
如何移植
===========================================

本指南说明"如何移植 NuttX"。

在本指南中，移植的范围是添加新的 SoC 或板目录，
使其可以构建并具有可启动到 NuttShell (NSH) 的工作示例。

移植的目标是确认 NuttShell(NSH) 的启动、"ostest" 的通过以及定时器。

将新的 SoC 和板移植到 NuttX 需要对 arch、SoC 和板目录进行修改。
有关详细信息，请参阅 :doc:`/quickstart/organization`、
:doc:`/components/arch/index`、:doc:`/components/boards`。

要正确移植 NuttX，我们必须理解启动序列和相关内核配置。
以下链接对此进行了解释，尽管这些依赖于特定的内核版本和配置。
（要理解它们，我们必须深入阅读代码。）

.. toctree::
  port_bootsequence.rst
  port_relatedkernelconfigrations.rst

关于实现，构建系统会告诉你 SoC/Board 目录的最小实现，
如果上游存在目标硬件 IP 的源代码，这些实现几乎可以通过
从其他 SoC/Board 目录复制粘贴来完成。
如果上游没有源代码，移植者必须自己实现。

移植步骤
=================

首先，你必须阅读并执行 :doc:`/quickstart/install` 和 :doc:`/quickstart/compiling_make`。
之后，尝试以下步骤。

+------+---------------+--------------------------------------------------------------------+
| 步骤 | 过程          | 说明                                                               |
+------+---------------+--------------------------------------------------------------------+
| 1    | 添加 SoC      |                                                                    |
+------+---------------+--------------------------------------------------------------------+
| 2    | 添加 Board    | 如果该板没有在市场上销售，                                         |
|      |               | 板目录应位于树外。                                                 |
|      |               | 详情请参阅 :doc:`/guides/customboards`。                           |
|      |               | 如果你想添加自己的应用，请参阅 :doc:`/guides/customapps`。         |
+------+---------------+--------------------------------------------------------------------+
| 3    | 配置          | 配置需要理解相关的内核配置。                                       |
|      |               | 请参阅 :doc:`/guides/port_relatedkernelconfigrations`。            |
+------+---------------+--------------------------------------------------------------------+
| 4    | 编译          | 编译会告诉你 SoC/Board 目录中需要哪些源文件才能通过编译。          |
+------+---------------+--------------------------------------------------------------------+
| 5    | 链接          | 链接会告诉你 SoC/Board 目录中需要哪些符号才能通过链接。            |
+------+---------------+--------------------------------------------------------------------+
| 6    | 实现          | 实现链接所需的符号。                                               |
+------+---------------+--------------------------------------------------------------------+
| 7    | 验证          | 执行 "apps/testing/ostest"。                                       |
|      |               | 我认为 ostest 的通过是正确移植的证明之一。                         |
|      |               | 并检查定时器实现，确认内核能否准确计时。                           |
+------+---------------+--------------------------------------------------------------------+
